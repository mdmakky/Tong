৩.৫ MongoDB — Messages Collection
messages (MongoDB Collection — Schema)
	{
	  _id: ObjectId,
	  conversation_id: String,        // PostgreSQL conversation.id অথবা group.id
	  conversation_type: String,      // "direct" | "group" | "private"
	  sender_id: String,              // PostgreSQL user.id
	  message_type: String,           // "text" | "image" | "video" | "audio" | "file" | "sticker" | "location" | "reply" | "forwarded"
	  content: {
	    text: String,                 // টেক্সট বা ক্যাপশন
	    media_url: String,            // Cloudinary URL
	    media_type: String,           // MIME type
	    media_size: Number,           // bytes
	    thumbnail_url: String,        // ভিডিওর থাম্বনেইল
	    file_name: String,            // ফাইলের নাম
	    duration: Number,             // অডিও/ভিডিওর দৈর্ঘ্য (seconds)
	    location: { lat, lng, name }, // লোকেশন শেয়ার
	    is_encrypted: Boolean,        // E2E?
	    encrypted_content: String,    // encrypted payload
	  },
	  reply_to: ObjectId,             // কোন মেসেজের রিপ্লাই
	  forwarded_from: ObjectId,       // কোথা থেকে ফরোয়ার্ড
	  reactions: [{
	    user_id: String,
	    emoji: String,
	    reacted_at: Date
	  }],
	  read_receipts: [{
	    user_id: String,
	    read_at: Date
	  }],
	  delivered_to: [String],         // delivered user_ids
	  edited_at: Date,
	  edit_history: [{ content, edited_at }],
	  deleted_for: [String],          // user_ids যাদের জন্য ডিলিট
	  is_deleted_for_all: Boolean,
	  pinned_by: String,
	  pinned_at: Date,
	  created_at: Date,
	  ttl: Date,                      // Auto-delete time (Disappearing messages)
	}

৪. ফিচার রিকোয়ারমেন্ট (Phase 1)
৪.১ ইউজার অথেন্টিকেশন ও অ্যাকাউন্ট
৪.১.১ রেজিস্ট্রেশন
* ইমেইল + পাসওয়ার্ড দিয়ে রেজিস্ট্রেশন (Manual — bcrypt হ্যাশ)
* ফোন নম্বর + OTP দিয়ে রেজিস্ট্রেশন (Resend Email OTP)
* ইমেইল ভেরিফিকেশন — ৬ সংখ্যার OTP (১০ মিনিট মেয়াদ, Redis-এ সংরক্ষিত)
* Username অটো-সাজেশন ও ইউনিক চেক
* Password strength indicator


৪.১.২ লগইন ও সিকিউরিটি
* JWT Access Token (15 মিনিট) + Refresh Token (7 দিন)
* Two-Factor Authentication (2FA) — TOTP (Google Authenticator)
* Device management — সকল লগইন ডিভাইস দেখা ও লগআউট
* Rate limiting — ব্রুট ফোর্স প্রতিরোধ
* IP-based suspicious login notification
* "Remember me" অপশন
* পাসওয়ার্ড রিসেট (Email OTP)


৪.২ ইউজার প্রোফাইল
* প্রোফাইল ছবি আপলোড ও ক্রপ করার সুবিধা
* Display name, username, bio আপডেট
* অনলাইন স্ট্যাটাস কাস্টমাইজেশন — Online / Away / Busy / Invisible
* Custom Status Message (যেমন: "Meeting-এ আছি")
* Last seen কে কারা দেখতে পাবে — সেটিং
* প্রোফাইল ছবি কে দেখতে পাবে — সেটিং (Everyone / Contacts / Nobody)
* লিংকড অ্যাকাউন্ট (Google, GitHub)
* অ্যাকাউন্ট ডিলিট অপশন (30 দিন grace period)


৪.৩ One-to-One চ্যাট
* রিয়েল-টাইম মেসেজ পাঠানো ও গ্রহণ (Socket.io)
* Message Types: Text, Image, Video, Audio, File, Location, Sticker, GIF
* Delivered ✓ এবং Read ✓✓ receipt
* Typing indicator ("লিখছেন...")
* মেসেজ Edit করা (edit history সহ)
* মেসেজ Delete — "শুধু আমার কাছে" অথবা "সবার কাছে"
* মেসেজে Reply (quote করে উত্তর দেওয়া)
* মেসেজ Forward (একাধিক কনভার্সেশনে)
* মেসেজ Pin করা
* Emoji Reactions (👍❤️😂😮😢😡)
* মেসেজ Search (কনভার্সেশনের ভেতরে)
* Media Gallery — সব শেয়ারকৃত ছবি/ফাইল দেখা
* Disappearing Messages (Auto-delete: 1h / 24h / 7d / custom)
* মেসেজ Bookmark/Star করা
* User Block / Unblock
* Report ইউজার


৪.৪ গ্রুপ চ্যাট
৪.৪.১ গ্রুপ তৈরি ও ম্যানেজমেন্ট
* গ্রুপ তৈরি (নাম, ছবি, বিবরণ সহ)
* গ্রুপের ধরন: Public / Private / Secret
* ইনভাইট লিংক জেনারেট ও শেয়ার
* QR Code দিয়ে গ্রুপে যোগ দেওয়া
* সর্বোচ্চ ২৫৬ সদস্য (কনফিগারযোগ্য)

৪.৫ প্রাইভেট / এনক্রিপ্টেড চ্যাট
* End-to-End Encryption (E2EE) সমর্থন — Signal Protocol বা AES-256
* Secret Chat — শুধু সেই ডিভাইসে সীমাবদ্ধ (no cloud backup)
* Screenshot notification — অন্য পক্ষ স্ক্রিনশট নিলে সতর্ক করবে
* Self-destructing messages — টাইমার অনুযায়ী অটো-ডিলিট
* "Lock" ফিচার — নির্দিষ্ট চ্যাট PIN/fingerprint দিয়ে লক
* No message forwarding — সিক্রেট চ্যাট থেকে ফরোয়ার্ড করা যাবে না

৭. সিকিউরিটি রিকোয়ারমেন্ট
৭.১ Authentication Security
* JWT Secret Rotation — নিয়মিত কী পরিবর্তন
* Refresh Token Rotation — প্রতিবার ব্যবহারে নতুন Refresh Token
* Token Blacklisting — লগআউটে Redis-এ টোকেন বাতিল
* Account lockout — ৫ বার ব্যর্থ লগইনে ৩০ মিনিট লক
* CSRF Protection — SameSite cookie policy


৭.২ API Security
* Rate Limiting — প্রতি IP প্রতি মিনিটে সর্বোচ্চ ১০০ request
* Helmet.js — Secure HTTP headers
* CORS — শুধু অনুমোদিত domain থেকে request
* Input Validation — সকল ইনপুট sanitize ও validate
* SQL Injection প্রতিরোধ — Parameterized queries / ORM
* XSS Protection — মেসেজ content sanitize
* File Upload Validation — MIME type ও extension চেক


৭.৩ Data Privacy
* GDPR Compliance — ইউজার ডেটা এক্সপোর্ট ও ডিলিটের সুবিধা
* Password Storage — bcrypt (cost factor ≥ 12)
* Sensitive Data Encryption at Rest — DB encryption
* Private Chat — End-to-End Encryption (Signal Protocol)
* Audit Logs — সংবেদনশীল কার্যক্রমের লগ রাখা

* একই সময়ে ১০০ ইউজারের মেসেজ পাঠানো — কোনো মেসেজ হারানো যাবে না

* JWT টোকেন মেয়াদ শেষে Refresh Token কাজ করা
* Block করা ইউজার মেসেজ পাঠাতে না পারা




১৪. সারসংক্ষেপ ও পরামর্শ
tong প্রজেক্টের Phase 1 সফলভাবে সম্পন্ন করতে নিচের বিষয়গুলো সবচেয়ে গুরুত্বপূর্ণ:


1. MongoDB ব্যবহার করুন messages-এর জন্য এবং PostgreSQL ব্যবহার করুন users ও groups-এর জন্য — এই হাইব্রিড পদ্ধতি পরবর্তী ফেজগুলোতে সবচেয়ে কার্যকর হবে।
2. Socket.io namespace আলাদা রাখুন — /chat, /notifications, /calling (ভবিষ্যতের জন্য) — এতে কোড পরিষ্কার থাকবে।
3. Redis থেকে শুরু করুন — Online presence ও typing indicator-এর জন্য Redis অপরিহার্য এবং এটি পরে Cache ও Queue-এর কাজেও আসবে।
4. API প্রথম থেকেই Versioned রাখুন (/api/v1/...) — Mobile App আসলে পুরানো version support করা সহজ হবে।
5. Test-Driven Development (TDD) অনুসরণ করুন — বড় প্রজেক্টে পরে bug fix করা অনেক কঠিন হয়।
